## Feature 迭代（在已有功能基础上的平滑扩展）

**F-01 [Feature迭代]**  
在图书详情页为每本书增加「加入购物车」能力：未登录用户点击时跳转登录并带回原页面；登录用户可将指定数量（不超过库存）加入购物车，顶栏购物车图标显示实时件数，点击侧滑抽屉可查看封面缩略图、书名、单价、数量步进器与小计，支持勾选删除与清空，金额汇总保留两位小数，刷新页面后购物车数据不丢失。

**F-02 [Feature迭代]**  
扩展用户体系：在导航用户菜单中新增「个人中心」入口，展示注册时间与邮箱；支持修改邮箱（需校验格式且不可与他人重复）与修改密码（需输入旧密码、新密码及确认密码，前后端均做一致性校验）；保存成功后 Toast 提示，并在个人中心顶部同步展示最新邮箱，退出再登录仍生效。

**F-03 [Feature迭代]**  
强化图书列表页的检索体验：在现有搜索与分类筛选基础上，增加价格区间滑块、仅看有货开关、排序方式（价格升/降、上架时间、库存量）；筛选条件写入 URL 查询参数，分享链接可还原同样结果；切换条件时列表带骨架屏，分页重置为第一页，结果区展示当前生效的筛选标签且可逐项清除。

**F-04 [Feature迭代]**  
为图书详情页增加评分与短评：登录用户可对已购或任意书提交 1–5 星与 20–500 字评论（每人每书仅一条，可编辑不可重复新建）；详情页展示平均分、评分数与评论列表（分页）；管理员在后台管理页可删除不当评论；未登录用户仅可浏览评分，提交按钮引导登录。

**F-05 [Feature迭代]**  
在首页「为什么选择我们」区块下方新增「编辑推荐」轮播：后台可标记图书 `is_featured`，首页横向滑动展示封面、书名、现价与「查看详情」按钮；无推荐书时展示占位文案；管理员在现有后台表格增加「设为推荐 / 取消推荐」开关，切换后首页无需硬刷新即可通过重新进入首页看到变化。

**F-06 [Feature迭代]**  
扩展后台图书管理：表格支持多选批量删除（二次确认展示书名列表）、批量修改分类与批量调整库存（增减或设为固定值）；操作栏保留单条编辑删除；批量接口需事务保证全部成功或全部回滚；完成后表格自动刷新并 Toast 汇总成功条数。

**F-07 [Feature迭代]**  
图书详情与列表封面支持本地上传：管理员在新增/编辑弹窗中除 URL 外可选择图片文件，前端压缩预览后上传至后端静态目录并回写 `cover_image`；上传进度条、格式与大小限制（如 jpg/png ≤2MB）需有明确错误提示；保存后列表卡片、详情大图与后台缩略图均显示新封面。

**F-08 [Feature迭代]**  
实现「收藏夹」：任意图书卡片与详情页提供心形收藏按钮，登录用户可收藏/取消，导航栏增加「我的收藏」页以网格展示已收藏图书并支持取消收藏与跳转详情；收藏状态在列表与详情间同步；未登录点击收藏时跳转登录。

**F-09 [Feature迭代]**  
登录页增加「记住我」：勾选后会话 token 与用户信息在 localStorage 持久化 7 天，未勾选则使用 sessionStorage 关闭浏览器即失效；应用启动时自动静默拉取 `/auth/me` 恢复登录态，失败则清理本地凭证并跳转登录；顶栏用户名与管理员菜单权限需与恢复结果一致。

**F-10 [Feature迭代]**  
图书列表增加「网格 / 列表」视图切换，偏好存入 localStorage；列表视图每行展示更大封面、作者、出版社、分类标签、价格与库存状态；移动端默认网格，桌面端记住上次选择；切换动画平滑且不重复请求接口。

**F-11 [Feature迭代]**  
详情页底部增加「相关推荐」：按相同分类推荐最多 6 本（排除当前书，库存为 0 的排在后面）；横向卡片可点击进入对应详情；分类为空时按同作者推荐；区块标题与空状态文案需完整。

**F-12 [Feature迭代]**  
注册流程增加邮箱格式实时校验、用户名长度 3–20 与密码强度提示（至少 8 位且含字母与数字）；提交前前端拦截不合法表单，后端保持原有唯一性校验；注册成功自动登录并跳转首页，失败时在对应字段下方展示错误信息而非仅全局 Toast。

**F-13 [Feature迭代]**  
后台管理表格增加列排序（ID、价格、库存、创建时间点击表头切换升/降序）与列显示配置（可隐藏 ISBN、出版社等列，配置持久化到 localStorage）；导出当前筛选结果为 CSV（含表头，中文不乱码），下载文件名带日期。

**F-14 [Feature迭代]**  
全局增加轻量「公告条」：管理员在后台维护一条当前生效公告（标题+链接可选），未过期时在所有页面顶栏下方展示，可关闭且关闭状态仅当次会话有效；后台可预览、启用/停用、设置结束时间；停用后刷新任意页不再出现。

**F-15 [Feature迭代]**  
优化图书详情阅读体验：ISBN 旁增加一键复制与成功提示；简介超过 200 字默认折叠并可展开/收起；提供「打印摘要」按钮弹出仅含书名、作者、出版社、价格与简介前段的打印预览；分享按钮将书名与当前页链接写入剪贴板并 Toast 反馈。

**F-16 [Feature迭代]**  
顶栏搜索升级为全局快捷搜索：输入关键词下拉展示最多 8 条书名匹配，键盘上下选择、回车进入详情；无结果时提示去图书列表页查看更多；在图书列表页搜索框与全局搜索状态双向同步（通过路由 query）。

**F-17 [Feature迭代]**  
为普通用户增加「浏览历史」：最近浏览 20 本书记录于服务端（登录用户）或 localStorage（游客），在用户菜单进入「最近看过」页按时间倒序展示，可单条删除与一键清空；从详情返回列表后历史立即更新。

**F-18 [Feature迭代]**  
后台新增/编辑图书表单增加 ISBN 实时校验（格式 10/13 位）与重复 ISBN 保存前拦截提示；价格、库存输入限制为非负数；提交时若网络失败保留表单草稿到 sessionStorage，再次打开弹窗可恢复。

**F-19 [Feature迭代]**  
图书列表卡片增加库存紧张标签（库存 ≤5 且 >0 显示橙色「仅剩 N 本」）；分类筛选旁展示各分类图书数量徽章；分页器旁增加每页条数选择 12/24/48，变更后记住用户偏好。

**F-20 [Feature迭代]**  
布局底部 footer 扩展为可配置三栏（关于我们、联系方式、友情链接），内容来自后端 JSON 配置接口；管理员后台增加「站点设置」Tab 可编辑并保存，保存后刷新前台 footer 即时更新，无需改代码部署。

## 0-1 构建（完整业务模块）

**N-01 [0-1构建]**  
订单中心：承接购物车勾选结果进入结算流程，采集收货人姓名、手机号、省市区与详细地址并做格式校验；生成订单后写入明细行快照（书名、单价、数量），主单支持待支付、已支付、已取消、已完成等状态流转；下单时锁定并扣减库存，取消或超时未支付时回滚库存；用户侧提供结算确认页、订单列表（按状态分组 Tab）与订单详情（含物流备注占位）；管理员在后台按时间、用户、状态检索订单，可手动改状态并填写内部备注。

**N-02 [0-1构建]**  
优惠券：管理员配置满减券模板，含名称、使用门槛、减免金额、有效期、发放总量与每人限领次数；前台设领券中心，展示可领取、已抢光、即将过期的券卡片；用户领取后落入「我的优惠券」，按未使用、已使用、已过期分栏；结算页拉取当前订单可用券列表，选中后实时重算商品总额、优惠额与应付金额；后端校验券状态、适用范围与叠加规则，核销时写使用记录。

**N-03 [0-1构建]**  
作者专题：将作者沉淀为独立档案实体，维护姓名、国籍、代表作、生平简介与头像；前台 `/authors` 以字母或作品数量排序浏览作者名录，支持关键词模糊搜索；作者详情页聚合其全部在售图书，展示作品数、均价区间与简介全文；管理员后台维护作者档案并与图书作者字段建立关联或映射；首页以主题卡片引流至作者名录。

**N-04 [0-1构建]**  
出版社名录：出版社作为一级实体，含名称、成立年份、官网、简介与 Logo；图书录入时通过下拉绑定出版社，列表页增加出版社筛选项；前台 `/publishers` 以卡片墙浏览出版社，进入详情后分页展示该社书目并支持按价格、出版时间排序；后台提供出版社增删改查及关联图书数量统计，避免删除仍有关联书的出版社。

**N-05 [0-1构建]**  
站内消息：为用户建立收件箱模型，消息含类型（订单、系统、营销）、标题、摘要、正文、跳转链接与已读标记；顶栏铃铛展示未读角标，消息中心支持按类型筛选、单条已读、全部已读与删除；业务侧在订单状态变更时自动生成消息，管理员可面向全体或指定用户发送广播；消息列表支持下拉加载更多，保留最近一年的归档策略配置入口。

**N-06 [0-1构建]**  
图书对比：在列表与详情提供「加入对比」入口，对比篮上限 3 本且同书不可重复；浮动条展示已选数量与快捷进入对比页；`/compare` 以表格式并排展示封面、书名、作者、出版社、分类、定价、库存、出版信息与简介节选，缺货行高亮提示；支持调整对比顺序、移除单本与一键清空，对比篮在会话内持久化并在关闭页面前保持状态。

**N-07 [0-1构建]**  
积分商城：用户注册赠送欢迎积分，订单实付金额按规则累计积分并写入流水（来源、变动值、余额、关联订单号）；顶栏常驻积分余额，点击进入积分明细与兑换商城；商城上架若干虚拟权益商品（包邮券、满减券等），标明所需积分与库存，兑换时扣减积分并生成兑换单；管理员可查询用户积分、手工调账并强制填写调账原因，调账记录进入审计流水。

**N-08 [0-1构建]**  
预约到货：针对库存为零的图书开放预约登记，采集手机号、可选邮箱与期望通知方式；用户「我的预约」按图书维度查看排队序号、登记时间与状态（排队中、已通知、已取消）；后台预约管理支持按书目聚合查看排队深度、导出明细、批量标记已通知；登记成功后详情页按钮态变为已预约，同一用户对同一图书不可重复提交。

**N-09 [0-1构建]**  
营销活动：管理员创建营销活动，配置标题、副标题、Banner 图、富文本规则说明、关联图书及生效时间段；前台 `/events` 列表区分进行中、即将开始与已结束，卡片展示倒计时与主视觉；活动详情 `/events/:id` 以专题布局呈现活动故事区与专属书单网格，未到开始时间的活动仅展示预告态；后台支持活动上下线、复制上一场配置与关联书排序。

**N-10 [0-1构建]**  
图书标签：标签与分类解耦，支持颜色、别名与启用状态；一本书可挂载多个标签，详情页以 chips 展示；列表页侧边栏提供标签多选筛选（交集 AND），热门标签在首页以胶囊入口直达筛选结果；后台维护标签字典、批量为图书打标或撤标，并统计每个标签下的书目数量与动销占比。

**N-11 [0-1构建]**  
客服工单：用户按咨询类型（订单、售后、账户、其他）提交工单，填写问题描述与可选附件链接，系统生成唯一工单号；「我的工单」列表展示状态（待处理、处理中、已关闭）与最近回复时间，详情页以时间线呈现用户与客服往来记录；管理员后台按状态、类型、用户检索工单，可回复、变更优先级、关闭工单并追加内部备注；新建回复时向用户推送站内消息提醒。

**N-12 [0-1构建]**  
礼品订单：结算流程增加礼品包装选项，勾选后展示包装费说明并开放赠言输入（字数上限与敏感词占位校验）；订单模型扩展礼品标记、赠言正文与包装费字段；用户订单详情以独立区块展示礼品信息与赠言；管理员订单详情同步展示礼品字段，便于出库打印；包装费计入订单应付总额，与普通商品行分开展示。

**N-13 [0-1构建]**  
主题书单：登录用户可创建书单，设置名称、封面、简介与公开/私密可见性；书单详情 `/lists/:id` 以封面墙排列书目，支持拖拽排序、单本备注与移除；向书单加书时可从全局搜索挑选；用户主页 `/u/:username` 展示其公开书单列表；私密书单仅创建者可见与编辑，公开书单允许访客浏览但不可改。

**N-14 [0-1构建]**  
到货订阅：用户按 ISBN 或图书详情发起邮箱订阅，登记邮箱、书目信息与订阅时间；「我的订阅」管理订阅项，支持取消与查看通知状态（待通知、已通知）；后台订阅列表按书目、ISBN、邮箱检索，提供模拟发送通知操作将状态推进为已通知；接口层保留邮件发送适配点，默认以日志或站内消息代替真实 SMTP，状态机需覆盖重复订阅与取消后再订阅。

**N-15 [0-1构建]**  
限时秒杀：管理员为指定图书配置秒杀价、活动库存、开始与结束时间，可与日常库存共用或独立池；前台 `/flash-sale` 分区展示抢购中、即将开始与已结束场次，抢购中显示倒计时与进度条（已售/秒杀库存）；下单结算识别秒杀价并校验时段，超卖保护以秒杀库存为准；场次结束后入口自动归入历史区，后台可复盘每场成交件数。

**N-16 [0-1构建]**  
门店自提：维护自提点档案（名称、地址、联系电话、营业时间、临时停业标记）；下单时用户选择快递配送或自提，自提订单生成六位取货码与二维码占位；订单详情展示自提点地图文案、营业提示与取货码；用户订单列表筛选自提单，状态涵盖待备货、待取货、已取货；后台维护自提点并可将订单标记为已取货，释放异常占用库存。

**N-17 [0-1构建]**  
阅读挑战：运营可配置挑战周期（默认 30 天）、每日打卡文案与规则说明；用户报名后进入挑战主页，日历视图展示每日打卡态、连续天数与累计打卡数；支持当日打卡并可选填阅读备注或页数，提供有限次数补签（消耗积分或次数池）；断签超过规则天数则挑战失败，可重新报名；个人中心提供挑战历史与成就徽章占位。

**N-18 [0-1构建]**  
专题落地页：运营配置一期「出版社专题周」或主题营销落地页，自定义主题色、头图、导语、分区文案与推荐图书序列；路由 `/spotlight` 以故事化版面呈现（首屏、书墙、编辑推荐、延伸阅读分区），区别于普通列表栅格；数据库仅允许一条当前生效配置，切换后整体换肤、文案与书单联动更新；后台提供可视化预览与定时生效时间。

**N-19 [0-1构建]**  
发票管理：用户在个人中心维护多条发票抬头（个人/企业、抬头名称、税号、开户行与地址电话可选），设默认抬头；下单时勾选是否需要发票并选择抬头类型，电子发票邮箱可选填；订单持久化开票快照，详情展示发票摘要；企业抬头校验税号长度与字符集，个人抬头隐藏税号字段；后台订单列表增加「需开票」筛选项与抬头信息列。

**N-20 [0-1构建]**  
多仓库存：每本图书在华东仓、华南仓分别维护库存数字，详情页展示分仓明细与合计；列表页继续展示可售总量，点击进入详情可看分仓；下单默认按「优先就近仓」或「库存最多仓」策略扣减（策略可配置），订单记录实际扣减仓；后台提供仓间调拨单，填写调出仓、调入仓与数量，调拨完成后两仓库存联动变更并记调拨日志。

**T-01 [代码测试]**  
使用 pytest 为 `backend/routes/books.py` 编写接口测试，覆盖分页、关键词搜索、分类筛选、ISBN 重复创建失败与未认证访问写接口被拒绝等场景，并提供可一键运行的测试配置与夹具初始化测试库。

**T-02 [代码测试]**  
为前端 `Books.vue` 与 `api/index.ts` 编写 Vitest 单元测试，mock 接口验证搜索触发、分类变更重置页码、空列表与错误提示分支，并配置 `npm run test` 脚本可在 CI 中执行。

**T-03 [代码测试]**  
编写认证链路集成测试：注册新用户、登录获取 token、携带 token 访问 `/api/auth/me` 与管理员图书创建接口的权限差异，测试数据库隔离且不与生产 seed 冲突。

## 代码重构（一句话投喂版）

**R-01 [代码重构]**  
将 `backend/routes/books.py` 中查询、校验与持久化逻辑下沉到独立 service 层，路由仅负责参数与 HTTP 响应，保持对外 API 行为不变并补充必要类型注解与日志点。

**R-02 [代码重构]**  
抽取前端重复图书卡片为共享组件，统一封面失败占位与价格/库存展示逻辑，优化 `Books.vue` 与 `Home.vue` 列表渲染（如稳定 key、减少不必要的响应式开销），视觉效果与现网一致。

## 工程化（一句话投喂版）

**E-01 [工程化]**  
新增 GitHub Actions 工作流：后端安装依赖并运行 pytest，前端执行 `npm ci` 与 `npm run build`，在 PR 与 main 推送时触发，失败时输出清晰日志摘要。

**E-02 [工程化]**  
将 JWT 密钥、数据库路径、CORS 来源改为环境变量注入，提供 `.env.example` 与 docker-compose 对齐配置，优化前后端 Dockerfile 多阶段构建以减小镜像体积且保持 `docker compose up` 一键可用。
