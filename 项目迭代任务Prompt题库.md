# 项目迭代任务 Prompt 题库

## Feature 迭代

1. 【Feature迭代】为现有图书列表页增加“价格区间 + 库存状态 + 排序”组合筛选能力：用户可以在列表页按最低价、最高价、是否有库存、最新上架、价格升降序进行筛选和排序，筛选条件需要同步到 URL 查询参数，刷新页面后保持当前状态；后端图书列表接口需要支持这些参数并返回准确分页结果，前端要在空结果、加载中、清空筛选、切换分页时都有清晰反馈，最终可以通过浏览图书列表页直接验证筛选、排序、分页是否一致。

2. 【Feature迭代】在图书详情页扩展“相关推荐”区域：根据当前图书的分类、作者、出版社三个维度推荐最多 6 本相关图书，排除当前图书，并在推荐卡片中展示封面、书名、价格和库存状态；后端需要提供推荐接口或在详情接口中返回推荐数据，前端需要处理推荐为空、封面加载失败、点击推荐书跳转并重新加载详情的情况，用户可以在任意图书详情页验证推荐内容是否随当前图书变化。

3. 【Feature迭代】为后台图书管理表格增加“批量操作”能力：管理员可以勾选多本图书，批量调整分类、批量修改出版社、批量上下架或批量删除，执行前需要二次确认，执行后表格数据、分页总数和提示信息要及时刷新；后端需要设计批量更新和批量删除接口，并校验管理员权限，前端需要在未选择数据、部分数据失败、操作完成后保留合理筛选条件等场景下可验证。

4. 【Feature迭代】将后台新增/编辑图书表单扩展为“实时封面预览 + 图片地址校验”：管理员输入封面 URL 后，右侧即时显示预览图，地址不可访问时显示默认封面和错误提示；保存前需要校验 URL 格式和必填字段，编辑已有图书时需要正确回显封面，保存成功后列表和详情页都能看到新封面效果，整个流程可以在后台管理页完成验证。

5. 【Feature迭代】为现有登录注册流程增加“个人资料页”：登录用户可以查看用户名、邮箱、注册时间和角色，并能修改邮箱与密码；修改密码需要输入旧密码，新密码要满足长度规则，修改成功后保持登录状态或按安全策略重新登录；前端需要新增入口和资料页面，后端需要提供当前用户资料更新接口，普通用户和管理员都可以在页面上验证不同角色展示。

6. 【Feature迭代】在图书列表和详情页增加“收藏图书”功能：登录用户可以收藏或取消收藏一本书，列表卡片和详情页都要显示收藏状态，未登录用户点击收藏时跳转登录并在登录后回到原页面；后端需要维护用户与图书的收藏关系并提供收藏列表接口，前端需要新增“我的收藏”页面用于查看收藏图书、取消收藏和跳转详情。

7. 【Feature迭代】为图书详情页增加“读者评分与短评”功能：登录用户可以给图书打 1-5 分并提交一条短评，同一用户对同一本书只能保留一条评价但允许编辑，详情页展示平均分、评价数量和最近评价列表；后端需要实现评价的创建、更新、分页查询和权限校验，前端需要支持评分控件、评论输入、编辑入口、空评价状态和提交后的即时刷新。

8. 【Feature迭代】在后台管理中增加“低库存预警”视图：管理员可以设置低库存阈值，页面展示库存低于阈值的图书，并支持按分类、出版社筛选；每本书提供快捷补货入口，提交补货数量后库存立即更新并从预警列表中重新计算；后端需要提供低库存查询和库存调整接口，前端需要在后台页面验证阈值变化、补货成功、无预警数据等状态。

9. 【Feature迭代】为首页增加“运营精选书架”：首页展示管理员配置的精选图书分组，例如“新品上架”“编辑推荐”“热销候选”，每组最多展示 8 本书并支持拖拽或排序字段控制展示顺序；后台需要提供精选分组维护入口，前端首页需要按照配置渲染分组，图书下架或删除后不应出现在精选书架中，用户可以通过首页和后台两处验证配置生效。

10. 【Feature迭代】为图书搜索增加“搜索建议”体验：用户在列表页搜索框输入关键词时，前端防抖请求后端建议接口，展示匹配的书名、作者、出版社关键词，点击建议后自动执行搜索；后端需要根据已有图书数据生成去重建议并限制返回数量，前端需要处理输入过短、无建议、快速输入、清空搜索等情况，最终可在图书列表页直接验证联想搜索效果。

11. 【Feature迭代】在后台图书管理中增加“导出当前筛选结果”功能：管理员可以将当前搜索、分类、库存状态等筛选下的图书导出为 CSV，导出内容包含书名、作者、出版社、ISBN、价格、库存、分类和创建时间；后端需要提供导出接口并复用查询条件，前端需要提供导出按钮、下载状态和错误提示，用户可以通过页面筛选后下载文件并核对内容。

12. 【Feature迭代】为图书详情页增加“到货提醒”功能：当图书库存为 0 时，登录用户可以订阅到货提醒，管理员在后台补货后该订阅状态显示为已触发；前端需要在详情页展示订阅入口和状态，在个人中心展示我的到货提醒列表，后端需要维护订阅记录并在库存从 0 变为正数时更新提醒状态，页面可验证订阅、补货、状态变化全流程。

13. 【Feature迭代】为管理员增加“图书变更记录”功能：每次新增、编辑、删除图书时记录操作人、操作类型、变更字段、操作时间；后台图书详情或独立审计页面可以查看某本书的变更历史，并支持按操作者、操作类型筛选；后端需要在现有图书接口中写入审计记录，前端需要让管理员在页面上能追踪一本书从创建到修改的变化。

14. 【Feature迭代】优化图书分类体系为“分类管理”：现有分类字段从自由输入扩展为可维护的分类字典，管理员可以新增、重命名、禁用分类，新增或编辑图书时只能选择启用分类；列表页分类筛选只展示启用分类，已有图书关联禁用分类时仍能显示但不能再被新书选择，前后端都需要处理分类变更后的回显和筛选。

15. 【Feature迭代】为后台图书表格增加“列配置与密度切换”：管理员可以选择显示哪些列，例如封面、作者、出版社、ISBN、价格、库存、创建时间，并可以在紧凑、默认、舒适三种表格密度之间切换；配置需要保存在本地，下次进入后台自动恢复，前端需要保证列隐藏后操作列仍可用，后端无需改变数据结构但页面必须能完整验证配置效果。

16. 【Feature迭代】为图书列表增加“视图模式切换”：用户可以在卡片视图和列表视图之间切换，列表视图展示更多文本信息和库存标签，卡片视图保持当前封面展示；视图模式需要保存在本地，分页、搜索、分类筛选在两种视图下表现一致，移动端需要自动适配，最终通过前端页面验证切换、刷新保留、点击进入详情等行为。

17. 【Feature迭代】为用户注册流程增加“用户名与邮箱实时可用性检查”：注册页在用户输入用户名或邮箱后自动检查是否已存在，并在输入框旁展示可用、已占用或格式错误状态；后端需要提供轻量检查接口并避免泄露过多用户信息，前端需要防抖请求、处理网络错误、提交前再次校验，用户可以在注册页验证重复账号阻止提交。

18. 【Feature迭代】为后台新增“封面图片上传”替代纯 URL 输入：管理员可以选择本地图片上传到后端 static 目录，上传成功后自动填入封面地址并显示预览，同时仍允许手动输入外链；后端需要限制文件类型和大小并返回可访问 URL，前端需要展示上传进度、失败提示、替换图片、编辑图书时保留原图等状态。

19. 【Feature迭代】为图书详情页增加“库存变动可视化”：管理员在后台多次调整库存后，详情页为管理员展示最近库存变化时间线，包括调整前数量、调整后数量、操作人和备注；普通用户只看到当前库存状态，后端需要记录库存调整日志，前端需要在后台补货或改库存后能回到详情页验证时间线数据。

20. 【Feature迭代】为全站增加“最近浏览图书”能力：用户访问图书详情时记录浏览历史，未登录用户保存在本地，登录用户保存在后端并支持跨设备；首页或个人中心展示最近浏览的 10 本书，重复浏览只更新时间，删除图书后历史中不再展示无效项，前端需要支持清空历史、点击回到详情、登录后合并本地浏览记录。

## 0-1 模块

21. 【0-1】购物车模块：用户可以从图书列表页和详情页把有库存图书加入购物车，购物车页面包含封面、书名、作者、单价、数量、小计、库存提示、选中状态和总价汇总；支持数量步进调整、单项删除、批量删除、清空购物车、失效图书标记、库存不足提示和登录后跨页面保留购物车内容；后端按用户维度维护购物车项，加入和更新数量时校验图书状态与库存，前端在导航栏展示购物车数量并处理未登录跳转、重复加入合并数量、库存变化后的提示文案。

22. 【0-1】订单中心模块：用户从购物车勾选图书后进入确认订单页，填写收货人、电话、地址、备注并生成订单编号，提交后扣减库存并固化商品快照、购买数量、成交价格和订单总额；用户端提供我的订单列表、订单详情、按状态筛选、取消未处理订单等能力，管理端提供全部订单查看、状态流转、订单备注和发货信息维护；后端需要处理订单状态机、库存事务、用户数据隔离和管理员权限。

23. 【0-1】优惠券模块：管理员维护满减券的名称、说明、门槛金额、优惠金额、有效期、发放数量、每人限领次数、适用分类和启用状态；用户在领券中心查看可领取、已领取、已过期和已用优惠券，并在订单确认页选择符合当前商品和金额条件的优惠券抵扣；后端处理领取库存、重复领取、有效期判断、订单占用与使用记录，前端展示优惠券状态、不可用原因和抵扣后的金额变化。

24. 【0-1】作者档案模块：后台维护作者姓名、头像、简介、国家或地区、出生年份、代表作说明和展示状态，图书编辑表单支持关联一个或多个作者档案；前台提供作者列表和作者详情页，详情页展示作者资料、关联图书、作品数量和分类分布，图书详情页中的作者名称可以跳转作者页；后端提供作者维护、关联图书查询、作者搜索和删除前关联检查，前端处理作者为空、头像缺失和多作者展示。

25. 【0-1】出版社档案模块：管理员维护出版社名称、Logo、官网、所在地、简介、成立年份和启用状态，图书编辑时从出版社档案中选择关联对象；前台提供出版社详情页，展示出版社信息、旗下图书、分类占比和最近上架书籍，图书详情页出版社名称支持跳转；后端提供出版社维护、关联图书查询、名称唯一性校验和禁用规则，前端支持后台筛选、详情展示、Logo 回退和禁用出版社的回显。

26. 【0-1】站内消息模块：系统向用户发送订单状态、到货提醒、管理员公告和账号安全类消息，顶部导航展示未读数量，消息中心支持全部、未读、已读、按类型筛选、单条已读、全部已读和删除；管理员后台支持发布公告类消息，选择发送对象、消息类型、标题、正文和有效期；后端维护消息主体、接收人、阅读状态和删除状态，前端需要在不同入口统一显示未读数量和消息内容。

27. 【0-1】管理员仪表盘模块：后台首页展示图书总数、用户总数、订单数量、低库存数量、库存总价值、最近上架图书、最近订单、分类库存占比和销售趋势等关键指标；统计卡片支持跳转到对应管理页并带入筛选条件，图表支持按最近 7 天、30 天、90 天切换；后端提供聚合统计接口和时间范围参数，前端处理加载态、无数据态、图表自适应和指标刷新。

28. 【0-1】主题书单模块：管理员维护书单标题、简介、封面、展示状态、排序权重和包含图书，支持调整图书顺序、移除图书和设置推荐语；前台提供书单列表页和书单详情页，用户可以浏览主题介绍、书单内图书、推荐语和关联分类，并从书单跳转到图书详情；后端处理书单与图书的多对多关系、排序字段、上下架过滤和封面地址，前端处理空书单、封面缺失和移动端布局。

29. 【0-1】会员等级模块：用户根据累计有效订单金额或管理员手动调整获得普通、银卡、金卡等会员等级，不同等级拥有不同折扣、等级标识和升级说明；后台维护等级名称、升级门槛、折扣比例、权益描述、排序和启用状态，用户中心展示当前等级、累计消费、距离下一等级的差额和权益说明；后端计算用户等级并在订单中固化折扣，前端在图书详情、购物车和订单确认页展示会员价。

30. 【0-1】图书标签模块：管理员维护标签名称、颜色、说明、排序和启用状态，并在图书编辑表单中为一本书绑定多个标签；前台图书列表支持按标签筛选，详情页展示标签并支持点击进入同标签列表，后台图书表格支持按标签快速过滤；后端维护标签字典与图书标签关系，处理标签名称唯一性、禁用标签回显和批量绑定，前端支持多选、搜索标签和颜色展示。

31. 【0-1】活动专题模块：管理员维护活动名称、封面、起止时间、活动说明、参与图书、活动价、活动库存和展示状态；前台提供活动列表和活动详情页，展示未开始、进行中、已结束等状态，活动图书展示原价、活动价、剩余库存和限购说明；后端校验活动时间、活动价不能高于原价、同一图书活动冲突和活动库存扣减，前端处理倒计时、状态标签和活动结束后的购买限制。

32. 【0-1】客服反馈模块：用户提交反馈时选择问题类型、关联订单或图书、填写描述、联系方式并上传图片附件，用户中心展示我的反馈列表、详情、处理状态和客服回复；后台客服页面支持按状态、类型、提交时间筛选反馈，查看附件、回复用户、修改处理状态和追加内部备注；后端维护反馈主表、附件、回复记录和状态流转，前端处理附件预览、回复时间线和不同角色的操作权限。

33. 【0-1】阅读试读模块：管理员为图书维护试读章节，包含章节标题、正文内容、排序、公开状态和更新时间，支持富文本或长文本录入；图书详情页提供试读入口，用户可以阅读公开章节、切换目录、调整阅读字号和返回图书详情，后台支持章节编辑、排序、隐藏和预览；后端维护章节数据和图书关联，区分公开内容与管理端完整内容，前端处理无试读、长文本排版和目录定位。

34. 【0-1】库存盘点模块：管理员发起盘点任务，选择盘点范围、负责人、备注和需要盘点的图书，录入实际库存后系统计算账面库存、实际库存和差异数量；盘点任务包含草稿、盘点中、已确认、已取消等状态，确认后批量修正图书库存并生成盘点明细；后端保证盘点确认不可重复并记录操作人，前端提供盘点列表、任务详情、差异高亮、批量录入和历史记录查看。

35. 【0-1】采购入库模块：管理员维护采购单，填写供应商、采购日期、备注，并添加图书、采购数量、采购单价和预计到货时间；采购单支持草稿、待入库、已入库、已取消状态，确认入库后增加库存、记录成本并生成库存变动记录；后台提供采购单列表、详情、编辑、确认入库和按供应商筛选，后端处理状态流转、金额汇总和重复入库限制，前端展示采购明细和库存影响。

36. 【0-1】用户地址簿模块：登录用户维护多个收货地址，包含联系人、手机号、省市区、详细地址、地址标签和默认地址设置；订单确认页自动带出默认地址并支持切换、编辑和临时刷新，删除默认地址后系统要求重新指定默认项；后端按用户隔离地址数据并校验手机号和默认地址唯一性，前端提供地址列表、编辑弹窗、默认标识、地址选择面板和空地址引导。

37. 【0-1】图书对比模块：用户在列表页或详情页把最多 4 本图书加入对比栏，对比页按封面、书名、作者、出版社、分类、价格、库存、评分、标签和简介等字段横向展示差异；对比栏支持固定在页面底部、移除单本、清空、折叠和跳转对比页，对比数据可以保存在本地或登录账户；后端按图书 ID 列表返回标准化对比数据，前端处理重复添加、数量上限和失效图书。

38. 【0-1】开放 API Key 模块：管理员为第三方系统维护 API Key，配置名称、备注、启用状态、过期时间、访问范围、调用频率限制和允许 IP；后台展示 Key 列表、密钥复制、禁用、轮换、最近调用日志和失败原因；后端实现 Key 鉴权、只读图书查询接口、调用日志、频率限制和过期校验，前端处理密钥只展示一次、复制提示、日志筛选和风险状态标识。

39. 【0-1】站点公告模块：管理员发布公告，配置标题、内容、展示位置、开始时间、结束时间、置顶状态、优先级和目标用户类型；前台首页、图书列表页或详情页根据展示位置显示当前有效公告，用户可以关闭公告并在有效期内保留关闭状态；后端提供公告管理和前台公告查询，处理时间窗口、排序、启停状态和关闭记录，前端处理横幅、弹窗、列表公告等不同展示方式。

40. 【0-1】数据导入向导模块：管理员上传 CSV 文件批量导入图书，流程包含文件上传、字段映射、数据预览、错误修正、跳过问题行和确认导入；后端解析文件并校验 ISBN 唯一性、必填字段、价格库存格式、分类和出版社字段，返回可编辑的错误列表与导入统计；前端提供步骤条、字段映射表、预览表格、行级错误提示、导入进度、成功失败统计和导入记录入口。
